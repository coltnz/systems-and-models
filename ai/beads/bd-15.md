# bd-15 — Extraction/validator hardening (issue #4)

- **Blocks:** —

## Scope (4 items)
1. **parseDraft runtime enum validation** (`packages/extraction/src/openai-adapter.ts`): replace the
   unchecked `as AtomKind` / `as Predicate` / `as SupportState` casts with runtime validation against
   the allowed values. An out-of-enum value from a (hostile/garbage) model response should produce a
   clear error from `parseDraft` (e.g. `OpenAI extraction: invalid kind "fact"`) rather than slipping
   through to surface as an opaque 500 at `validatePack`.
2. **`minItems: 1` on atom `anchors`** in `buildResponseSchema` so the model cannot emit an atom with
   `anchors: []` (validates clean as `generated` but can never be `published` and can't be cited).
3. **Pack-global id uniqueness** (D-013):
   - **Validator** (`packages/validator/src/index.ts`): add a check that flags any id appearing in more
     than one of {sources, anchors, atoms, relationships, derivations} (new code
     `cross_collection_duplicate_id`, severity graph). Keep the existing per-collection `duplicate_id`.
   - **Assembly chokepoint** (`packages/extraction/src/draft.ts` `assembleExtraction`): guarantee
     extraction never emits a globally-colliding id. Build a `used` set seeded with the reserved ids
     (`sourceAssetId`, `derivationId`, and all `validAnchorIds`); drop (keep-first) any atom or
     relationship whose id is already in `used`; add kept ids to `used`. Keep a separate `atomIds` set
     of KEPT atom ids for the existing dangling-endpoint check. This preserves the no-stranded-draft
     property: the new validator check can only fire on external/hand-edited packs, never on extraction output.
4. **date-time contract guard** (D-014): keep the tz-aware rule graph-owned (do NOT change runtime
   severity). Add a regression test asserting the only `format:"date-time"` field in
   `spec/learning-pack.schema.json` is `DerivationRun.created_at` and that a zoneless/date-only value is
   rejected at the graph layer — so a future `date-time` field forces a conscious graph-check decision.

## Definition of done / tests
- parseDraft: an out-of-enum `kind`/`predicate`/`support_state` throws a clear error (test).
- response schema: atom `anchors` has `minItems:1` (test that schema requires it; existing mock/e2e unaffected).
- global uniqueness: validator flags an atom.id == anchor.id (or == relationship.id) cross-collection
  collision; assembly drops an atom/relationship whose id collides with a reserved/used id, and the
  assembled pack validates clean (no `cross_collection_duplicate_id`).
- date-time guard test passes; existing date-time tests unchanged.
- Full root gate green; committed `packages/e2e/examples/*.json` byte-unchanged and still validate.
