## Problem (reviewer, [P1])
`assembleExtraction` validates relationship *anchor* ids but does NOT verify that
`from_atom_id`/`to_atom_id` are among the atoms produced in the same draft, nor that a
present `anchor_ids` is non-empty. So an OpenAI draft can carry a relationship whose
endpoint is a non-existent atom (or an empty `anchor_ids`) → validator emits a graph error
(`dangling_relationship_atom_ref` / `relationship_empty_anchor_ids`) → `postPackDraft`
persists it (only blocks structural). The only later relationship mutation is accept/reject,
and rejecting does NOT remove the edge from validation → the reviewer is stuck with a draft
that can **never** be saved reviewed (409 forever). bd-11 #1 fixed empty `anchor_ids` only in
OpenAI `parseDraft`; the dangling-endpoint case and the shared-layer guarantee remain.

## Fix — make `assembleExtraction` always emit a graph-ref-clean draft
In `packages/extraction/src/draft.ts` `assembleExtraction` (the shared chokepoint for BOTH
adapters):
1. Compute the assembled atom-id set. **DROP** (skip) any draft relationship whose
   `from_atom_id` or `to_atom_id` is not in that set — a relationship can't reference an atom
   outside the draft, and such an edge is unrepairable through the review UI.
2. **Normalize empty `anchor_ids`**: if a relationship's `anchor_ids` is present but empty,
   emit a no-anchor relationship (omit `anchor_ids` + `support_state`). (Authoritative here,
   regardless of adapter; keeps bd-11's parseDraft fix as belt-and-suspenders.)
3. KEEP the existing `UnknownAnchorError` throw for a NON-empty `anchor_ids` that references an
   id not in `validAnchorIds` (the "never invent anchors" hard guarantee — unchanged; its test stays).
4. The derivation's `output_ids` must list only the KEPT relationships (plus the atoms).

Net: a draft assembled from any adapter output is always free of relationship-caused graph
errors, so `postPackDraft` can never persist a draft that blocks reviewed-save.

## Definition of done / tests (packages/extraction/src/index.test.ts)
- A draft relationship whose `from`/`to` atom is not among the draft atoms is DROPPED (absent
  from `result.relationships` and from the derivation `output_ids`); remaining valid edges kept.
- A relationship with empty `anchor_ids` assembles to a no-anchor relationship.
- The assembled pack validates (no `dangling_relationship_atom_ref` / `relationship_empty_anchor_ids`)
  even when given a deliberately-bad relationship.
- The existing "throws UnknownAnchorError for an invented (non-empty, unknown) anchor" behavior + test still pass.
- Mock-path output unchanged (mock edges are valid) → committed `examples/*.json` unchanged & still validate.
