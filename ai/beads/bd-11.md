## Problem
A high-effort review (7 finder angles + verification) of the merged alpha found a small
set of confirmed correctness issues — none break the offline demo, but #1 bites the first
real OpenAI run. Fix the confirmed ones; the rest are accepted-by-design (recorded below).

## Fixes (each with a test; keep all 5 gates green)
1. **[high, OpenAI path] empty `anchor_ids` → unsaveable pack.** In `packages/extraction/src/openai-adapter.ts`
   `parseDraft`, normalize an empty relationship `anchor_ids` array to **omit** `anchor_ids` (and
   `support_state`) so `assembleExtraction` emits a valid no-anchor relationship instead of one that
   trips the validator's `relationship_empty_anchor_ids` graph error (which `postPackDraft` persists and
   which then blocks `POST /packs/:id/reviewed` with a permanent 409). Test: a parsed draft with a
   `[]`-anchor relationship assembles to a relationship without `anchor_ids`, and the wrapped pack validates.
2. **[med] `authored_by` → "mixed" on human edit.** In `packages/server/src/index.ts` `patchAtom` `op:"edit"`,
   if `atom.authored_by === "ai"` set it to `"mixed"` (a human has now co-authored it). Test asserts it.
3. **[med] thread the server clock into extraction.** Make `ServerOptions.now` cover the whole data path:
   pass the server's `now` to the extraction so the `extract` derivation's `created_at` is deterministic.
   Cleanest: add an optional `now?: () => Date` to `ExtractionInput` (adapters use `input.now ?? this.now ??
   default`) and have the server pass `deps.now`. Test: `createServer({now})` → draft's extract derivation
   `created_at` equals the injected instant.
4. **[med] split-lite provenance.** In `splitAtom`, point the new atom's `derivation_id` at the **edit**
   derivation created for the split, and include the new atom id in that derivation's `output_ids`. (Have
   `appendEditDerivation` return the derivation/id.) Test: after split, the new atom's `derivation_id`
   resolves to a derivation whose `output_ids` contains it.
5. **[low] traversable/eligible consistency.** In `packages/validator/src/index.ts`, make `traversableEdges`/
   `isTraversable` accept `review_state ∈ {reviewed, published}` (published ⊇ reviewed; matches the tutor's
   `isEligible`). Test asserts a `published` edge is traversable.
6. **[low] surface 422 validation in the web UI.** In `packages/web/src/App.tsx` `run()`, when an `ApiError`
   carries a `body.validation`, update the validation panel from it (and keep the message) instead of only
   showing the message. (No new server/route change.) Test asserts the panel renders the 422 validation.
7. **[cleanup] remove dead `void reviewedEdges`** in `packages/server/src/tutor.ts` (keep a one-line comment
   that retrieval is atom-centric and any future edge use must go through `traversableEdges`).
8. **[cleanup] drop the redundant media-type blocklist** in `packages/ingest/src/index.ts` (keep the
   `!== 'markdown' && !== 'text'` allowlist check + `UnsupportedMediaTypeError`).
